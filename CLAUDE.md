# プロジェクト目的

美容室向けAI投稿・口コミ管理自動化システム。

# ユーザー

ユーザーはプログラミング初心者。
説明は日本語で行う。
専門用語には短い説明を付ける。
一度に1つの作業だけ進める。

# 開発ルール

- TypeScriptを使用する
- できるだけ単純な構成にする
- 機能ごとにファイルを分ける
- 入力値を必ず検証する
- エラーを無視しない
- 変更前に作業内容を説明する
- 変更後にテストを行う
- テストが失敗した状態で完了と報告しない
- 不明点を勝手に決めない
- 古いモデル名や廃止予定のAPIを固定しない

# 秘密情報

- APIキーをコードに書かない
- パスワードをコードに書かない
- .env.localをGitHubへ送らない
- ログへ秘密情報を出さない
- 顧客の電話番号や氏名をAIへ不要に送らない

# 外部操作

以下は実行前に必ず人間へ確認する。

- 外部サービスへの投稿
- 外部サービスへの返信
- 課金が発生する処理
- データ削除
- データベース構造の変更
- 本番環境への公開
- SALON BOARDへの操作
- アクセストークンの更新

# SALON BOARD連携方針

- SALON BOARDには公式APIが存在しないため、ブラウザ自動操作(スクレイピング)で連携する
- ログインID・パスワードは環境変数で保管し、コードには書かない
- 利用規約違反・アカウント停止・画面変更による故障のリスクを許容した上での判断であることを前提とする
- 実装は開発順序の第4段階で行う(第1段階では着手しない)
- 実際にSALON BOARDへログイン・投稿する操作は、都度人間へ確認する

# 口コミルール

- 星1〜3は無確認で返信しない
- 口コミにない事実を作らない
- 薬剤事故やケガは緊急扱い
- 返金や法的責任をAIが勝手に約束しない
- 個人情報を返信へ書かない

# 投稿ルール

- 同一画像の二重投稿を防止する
- 投稿前に文字数を確認する
- 誇大表現を確認する
- 医療効果を断定しない
- 投稿結果と外部投稿IDを保存する
